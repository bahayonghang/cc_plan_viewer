> A VSCode extension for reviewing and annotating Claude Code plan files — with inline comments, Mermaid diagrams, and project grouping.

## Overview

Plan Viewer reads the `.md` plan files generated by [Claude Code](https://claude.ai/code) (stored in `~/.claude/plans/` by default) and renders them inside VSCode with a rich review workflow: syntax-highlighted Markdown, live Mermaid diagrams, and a five-type inline commenting system that can optionally write comments back into the plan file itself.

## Features

### 📋 Sidebar Plan List

- Browse all plan files in a dedicated Activity Bar panel
- **Dynamic Colorful Icons**: Auto-assigned colorful icons based on plan name (Fix/Bug, Refactor, UI, Doc, Test...)
- Toggle OpenSpec ↔ Claude Plans views with quick switch icon
- Auto-refresh on file changes (300 ms debounce)
- Open any plan / artifact directly in the VSCode text editor

### 📄 Markdown Rendering

- Full Markdown support with syntax-highlighted code blocks (via [highlight.js](https://highlightjs.org/))
- Live [Mermaid](https://mermaid.js.org/) diagram rendering (flowcharts, sequence diagrams, Gantt charts, etc.)
- Configurable font size (10 – 24 px) and line height (1.2 – 2.5)

### 💬 Inline Comment System

Five comment types to cover every review scenario:

| Type | Icon | Use Case |
|------|------|----------|
| Comment | 💬 | General notes |
| Suggestion | 💡 | Propose an alternative |
| Question | ❓ | Ask for clarification |
| Approve | ✅ | Sign off on a section |
| Reject | ❌ | Mark a section as needing rework |

**Two ways to add a comment:**
1. **Section comment** — hover over any heading to reveal the comment button
2. **Text-selection comment** — select any text, then click the tooltip that appears

### 🗂️ Comment Panel

- Slide-out panel listing all comments for the current plan
- Click any comment to scroll the viewer to its section
- Delete comments directly from the panel

### 💾 Dual Comment Storage

Comments are persisted in two places simultaneously:

| Store | Key / Location | Notes |
|-------|----------------|-------|
| VSCode `globalState` | `planViewer.comments.<planId>` | Always active |
| Plan `.md` file | `## 📝 Review Comments` section | Controlled by `embedCommentsInMarkdown` setting |

A bidirectional sync runs on every plan load so both stores stay consistent.

## Installation

### From VSIX (Recommended)

1. Download the latest `plan-viewer-*.vsix` from the [Releases](https://github.com/anthropic-community/plan-viewer-vscode/releases) page
2. In VSCode: **Extensions → ··· → Install from VSIX…**
3. Select the downloaded file and reload VSCode

### From Source

```bash
git clone https://github.com/anthropic-community/plan-viewer-vscode.git
cd plan-viewer-vscode
just install   # npm install
just build     # build + package → outputs/plan-viewer-*.vsix
```

Then install the generated `.vsix` as above.

## Usage

1. Open VSCode — the **Plan Viewer** icon appears in the Activity Bar
2. Click it to open the plan list (reads `~/.claude/plans/` by default)
3. Click any plan to open it in the Webview panel
4. **To add a comment**: hover over a heading → click 💬, or select text → click the tooltip
5. **To review all comments**: click the comment counter in the toolbar to open the Comment Panel

## Configuration

Open **Settings** (`Ctrl+,`) and search for `planViewer`, or click the gear icon in the Plan Viewer panel header.

| Setting | Type | Default | Description |
|---------|------|---------|-------------|
| `planViewer.plansDirectory` | `string` | `""` | Custom path to the plans directory. Leave empty to use `~/.claude/plans` |
| `planViewer.groupByProject` | `boolean` | `true` | Group plans by project in the sidebar |
| `planViewer.embedCommentsInMarkdown` | `boolean` | `true` | Write comments back into the `.md` file under `## 📝 Review Comments` |
| `planViewer.fontSize` | `number` | `14` | Webview font size in px (10 – 24) |
| `planViewer.lineHeight` | `number` | `1.7` | Webview line height (1.2 – 2.5) |

## Commands

All commands are available via the Command Palette (`Ctrl+Shift+P`) under the **Plan Viewer** category:

| Command | Description |
|---------|-------------|
| `Plan Viewer: Open Plan` | Open a plan in the Webview |
| `Plan Viewer: Refresh Plans` | Reload the plan list |
| `Plan Viewer: Open in Editor` | Open the current plan `.md` in the text editor |
| `Plan Viewer: Settings` | Jump to Plan Viewer settings |
| `Plan Viewer: Toggle Project Grouping` | Switch between flat and grouped sidebar |
| `Plan Viewer: Expand All Groups` | Expand all project groups |
| `Plan Viewer: Collapse All Groups` | Collapse all project groups |

## Architecture

```
plan-viewer-vscode/
├── src/
│   ├── extension.ts              # Activation & command registration
│   ├── types.ts                  # Shared interfaces
│   ├── services/
│   │   ├── planService.ts        # List/load plans, project extraction, cache
│   │   ├── commentService.ts     # Add/delete comments, dual-write logic
│   │   ├── commentSync.ts        # Bidirectional JSON ↔ Markdown sync
│   │   ├── commentBuilder.ts     # Comment → Markdown block
│   │   ├── commentParser.ts      # Markdown block → Comment
│   │   ├── commentInjector.ts    # Inject/remove blocks in .md files
│   │   └── fileWatcher.ts        # Auto-refresh (300 ms debounce)
│   ├── providers/
│   │   ├── planTreeProvider.ts   # TreeDataProvider + project grouping
│   │   ├── planTreeItem.ts       # Individual plan TreeItem
│   │   └── webviewPanelManager.ts# Singleton Webview lifecycle & message routing
│   └── webview/                  # Preact frontend (separate Vite build)
│       ├── App.tsx
│       ├── lib/messageProtocol.ts
│       └── components/           # MarkdownViewer, CommentPanel, Toolbar …
└── l10n/
    ├── bundle.l10n.json          # English strings
    └── bundle.l10n.zh-cn.json   # Chinese strings
```

Two independent build processes communicate via a typed `postMessage` protocol:

- **Extension Host** (`src/` → `dist/extension.js`) — Node.js, full VSCode API access
- **Webview** (`src/webview/` → `dist-webview/`) — sandboxed browser iframe, Preact + Vite
